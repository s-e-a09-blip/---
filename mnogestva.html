Множества

Что такое множество в Python?

Множества (тип set) — это неупорядоченные коллекции уникальных элементов. Они хранят только неповторяющиеся значения и не поддерживают индексацию. Множества нужны для операций над множествами: объединение, пересечение, разность, а также для быстрой проверки наличия элемента и удаления дубликатов.

Теперь, когда мы знаем, что такое множества, давайте научимся с ними работать. Python предлагает множество удобных операций для манипуляции множествами.

Создание множества

Множества создаются с помощью фигурных скобок {} или функции set(). Обратите внимание, что дубликаты автоматически удаляются. 

Программный код:

      fruits = {"яблоко", "банан", "апельсин"}
      numbers = set([1, 2, 3, 3, 4, 4, 5])  # Из списка с дубликатами
      print(fruits)
      print(numbers)

📝 Пояснение: Первое множество создаётся напрямую с помощью фигурных скобок. Второе — из списка, который содержит повторяющиеся элементы 3 и 4. Функция set() автоматически удаляет дубликаты, оставляя только уникальные значения. Множества выводятся в произвольном порядке, так как они неупорядочены. 

Вывод:

              {'банан', 'яблоко', 'апельсин'}
              {1, 2, 3, 4, 5}

Добавление элементов (add и update)

Метод add() добавляет один элемент, а update() добавляет несколько элементов из другой коллекции (списка, кортежа или другого множества). 

Программный код:

      colors = {"красный", "синий"}
      colors.add("зеленый")              # Добавляем один элемент
      colors.update(["желтый", "фиолетовый"])  # Добавляем несколько элементов
      print(colors)

📝 Пояснение: Сначала метод add() добавляет в множество цвет "зеленый". Затем метод update() принимает список с двумя цветами и добавляет их оба. Если бы мы попытались добавить уже существующий элемент, он бы просто проигнорировался, так как множество хранит только уникальные значения. 

Вывод:

              {'желтый', 'синий', 'зеленый', 'фиолетовый', 'красный'}

Удаление элементов (remove, discard и pop)

remove() удаляет элемент (если его нет — вызовет ошибку KeyError), discard() удаляет элемент (если его нет — ничего не делает), pop() удаляет и возвращает произвольный элемент (так как множество неупорядочено). 

Программный код:

      animals = {"кот", "собака", "птица", "рыба"}
      animals.remove("птица")           # Удаляем "птица" - существует
      animals.discard("змея")          # Пытаемся удалить "змея" - её нет, но ошибки не будет
      random_item = animals.pop()      # Удаляем случайный элемент
      print(animals)
      print("Удаленный элемент:", random_item)

📝 Пояснение: remove() удаляет указанный элемент, но если его нет — программа завершится с ошибкой. discard() безопаснее — он просто ничего не сделает, если элемента нет. pop() полезен, когда нужно получить и удалить любой элемент множества, но помните: вы не можете контролировать, какой именно элемент будет удалён. 

Вывод:

              {'собака', 'рыба'}
              Удаленный элемент: кот

Объединение множеств (union или |)

Объединение возвращает новое множество, содержащее все элементы из обоих исходных множеств. Дубликаты автоматически удаляются. 

Программный код:

      set1 = {1, 2, 3}
      set2 = {3, 4, 5}
      result = set1.union(set2)      # Метод union
      result2 = set1 | set2         # Оператор | (более краткая запись)
      print("Метод union:", result)
      print("Оператор | :", result2)

📝 Пояснение: Оба способа делают одно и то же. Элемент 3 присутствует в обоих множествах, но в результате он появляется только один раз, так как множество не хранит дубликаты. Оператор | — это более короткий и удобный способ записи объединения. 

Вывод:

              Метод union: {1, 2, 3, 4, 5}
              Оператор | : {1, 2, 3, 4, 5}

Пересечение множеств (intersection или &)

Пересечение возвращает новое множество, содержащее только те элементы, которые присутствуют в обоих исходных множествах. 

Программный код:

      set1 = {1, 2, 3, 4}
      set2 = {3, 4, 5, 6}
      result = set1.intersection(set2)   # Метод intersection
      result2 = set1 & set2              # Оператор & 
      print("Метод intersection:", result)
      print("Оператор & :", result2)

📝 Пояснение: Пересечение находит общие элементы. В этом примере числа 3 и 4 есть в обоих множествах, поэтому они попадают в результат. Оператор & работает аналогично оператору пересечения в математике. 

Вывод:

              Метод intersection: {3, 4}
              Оператор & : {3, 4}

Разность множеств (difference или -)

Разность возвращает элементы первого множества, которых нет во втором множестве. 

Программный код:

      set1 = {1, 2, 3, 4, 5}
      set2 = {4, 5, 6, 7}
      result = set1.difference(set2)   # Метод difference
      result2 = set1 - set2          # Оператор -
      print("Метод difference:", result)
      print("Оператор - :", result2)

📝 Пояснение: Из первого множества вычитаются все элементы, которые есть во втором. В результате остаются только те элементы, которые уникальны для первого множества. Обратите внимание: операция не симметрична — set1 - set2 и set2 - set1 дадут разные результаты. 

Вывод:

              Метод difference: {1, 2, 3}
              Оператор - : {1, 2, 3}

Симметрическая разность (symmetric_difference или ^)

Симметрическая разность возвращает элементы, которые есть только в одном из множеств, но не в обоих одновременно. 

Программный код:

      set1 = {1, 2, 3, 4}
      set2 = {3, 4, 5, 6}
      result = set1.symmetric_difference(set2)  # Метод
      result2 = set1 ^ set2                     # Оператор ^
      print("Метод symmetric_difference:", result)
      print("Оператор ^ :", result2)

📝 Пояснение: Симметрическая разность — это объединение минус пересечение. В результат попадают элементы, которые есть в первом или во втором множестве, но не в обоих сразу. В нашем примере 1 и 2 есть только в set1, 5 и 6 — только в set2, а 3 и 4 есть в обоих, поэтому они не попадают в результат. 

Вывод:

              Метод symmetric_difference: {1, 2, 5, 6}
              Оператор ^ : {1, 2, 5, 6}

Проверка подмножества и надмножества

issubset() проверяет, все ли элементы множества входят в другое множество (является ли оно подмножеством). issuperset() проверяет, содержит ли множество все элементы другого (является ли оно надмножеством). 

Программный код:

      set1 = {1, 2, 3}
      set2 = {1, 2, 3, 4, 5}
      print("set1 подмножество set2?", set1.issubset(set2))
      print("set2 надмножество set1?", set2.issuperset(set1))
      print("set1 надмножество set2?", set1.issuperset(set2))

📝 Пояснение: Все элементы set1 (1, 2, 3) присутствуют в set2, поэтому set1 является подмножеством set2, а set2 — надмножеством set1. Обратное неверно, так как в set2 есть элементы 4 и 5, которых нет в set1. 

Вывод:

              set1 подмножество set2? True
              set2 надмножество set1? True
              set1 надмножество set2? False

Проверка наличия элемента и длина множества

Оператор in проверяет, есть ли элемент в множестве (очень быстро, даже для больших множеств), а функция len() возвращает количество уникальных элементов. 

Программный код:

      fruits = {"яблоко", "банан", "апельсин"}
      print("Есть ли банан?", "банан" in fruits)
      print("Есть ли вишня?", "вишня" in fruits)
      print("Количество элементов в множестве:", len(fruits))

📝 Пояснение: Оператор in в множествах работает очень быстро — за O(1) в среднем. Это одно из главных преимуществ множеств перед списками, где поиск элемента занимает O(n). Функция len() возвращает количество уникальных элементов. 

Вывод:

              Есть ли банан? True
              Есть ли вишня? False
              Количество элементов в множестве: 3

Очистка множества и замороженные множества

clear() удаляет все элементы из множества, превращая его в пустое множество. frozenset() создает неизменяемую версию множества, которую можно использовать в качестве ключа словаря. 

Программный код:

      numbers = {1, 2, 3, 4, 5}
      numbers.clear()
      print("После очистки:", numbers)
      
      frozen = frozenset([1, 2, 3, 3, 4])
      print("Замороженное множество:", frozen)
      # frozen.add(5)  # Если раскомментировать — будет ошибка!

📝 Пояснение: clear() полностью опустошает множество. frozenset() — это неизменяемая версия множества. Его нельзя изменить (добавлять или удалять элементы), но он может использоваться в качестве ключа словаря или элемента другого множества, в отличие от обычного set. 

Вывод:

              После очистки: set()
              Замороженное множество: frozenset({1, 2, 3, 4})
